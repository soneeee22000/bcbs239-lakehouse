## BUILD RULES (Claude MUST follow these)

### Rule 1: PRD Is Law

- Every feature traces back to a user story in `docs/PRD.md`
- If a feature is not in the PRD, DO NOT build it — ask first
- If requirements are ambiguous, STOP and ask before writing code

### Rule 2: Build in Order

- Follow the milestone phases in PRD §7 strictly (Weekend 1 → Weekend 2)
- Complete the Weekend 1 gate before starting Weekend 2 work
- Each phase has a quality gate — ALL gate criteria must pass before proceeding

### Rule 3: Test Before Implement

- Write the test file FIRST for every new module
- Test should initially fail (red); minimum code to pass (green); refactor
- Untested code is unshipped code

### Rule 4: One Thing at a Time

- Implement ONE user story at a time (PRD §3 Stories 1-5)
- Each story follows: test → implement → verify → commit
- Commit after each completed story with a conventional commit
- **Never include AI co-author tags or "Generated by AI" footers** (per global CLAUDE.md)

### Rule 5: Refuse Killed Claims

- The killed-phrase set is in PRD Appendix B. NEVER include any of those in:
  - README copy
  - Lakeview dashboard captions
  - Loom / GIF script
  - Recruiter pitch
  - CV bullets
  - Cover letter
  - GitHub project description
  - Repo topics / tags
  - Python comments / SQL comments
- Approved framing only: "reference implementation of the BCBS 239 risk-data-aggregation lakehouse pattern Capgemini Risk Data Insights and Big-4 BCBS 239 advisory practices recommend G-SIBs build atop Databricks Unity Catalog."
- `tests/test_repo_hygiene.py::test_no_killed_phrasing` enforces.

### Rule 6: Synthetic Data Only — Always Disclaimed

- Real G-SIB data is OUT OF SCOPE forever (per PRD §8)
- Every claim about pipeline behaviour states "on synthetic data" explicitly
- Synthetic identifiers are obvious fakes (LEI prefix `9999`, names like "AcmeBank S.A.")
- README opening states the synthetic-only constraint within the first 200 words

### Rule 7: Types Are Mandatory

- All Python functions have type hints (strict mypy, no `Any` escape hatches)
- All synthetic data definitions go through Pydantic v2 models or dataclasses
- All DQ scorers return typed dataclasses, not raw dicts

### Rule 8: No Dead Code, No TODOs in Commits

- Remove all `print()` debug statements before committing
- No `# TODO` comments — implement now or add to PRD §9 Open Questions
- No commented-out code blocks
- No unused imports / variables

### Rule 9: AI Tool Artifact Hygiene

- `.claude/`, `.sixth/`, `.cursor/`, `.codex/`, `.copilot/` directories all gitignored
- Before any push: `git ls-files | grep -iE 'claude|codex|cursor|copilot'` should be empty
- Commits never reference AI assistance

### Rule 10: Databricks Resource Hygiene

- Every Databricks resource (UC catalog, schema, Lakeview dashboard, workflow) is provisioned via an **idempotent script** in `notebooks/` or `src/bcbs239_lakehouse/lineage/`
- No manual UI clicks documented as setup — everything reproducible via `make`
- Cleanup script `make uc-teardown` reverses all UC provisioning

### Killed-Phrase Anti-Regression Test Pattern

```python
# tests/test_repo_hygiene.py
import pathlib
import pytest

KILLED_PHRASES = [
    "replaces collibra",
    "replaces alation",
    "replaces atlan",
    "replaces capgemini",
    "replaces big-4",
    "production-ready bcbs 239 evidence layer",
    "production-grade for regulator review",
    "big-4 alternative",
]

REPO_ROOT = pathlib.Path(__file__).parent.parent

@pytest.mark.parametrize("phrase", KILLED_PHRASES)
def test_no_killed_phrasing(phrase: str) -> None:
    """No artifact may reintroduce the moat-check killed-claim phrasing."""
    hits: list[str] = []
    for path in REPO_ROOT.rglob("*"):
        if path.suffix not in {".md", ".sql", ".py"}:
            continue
        if "node_modules" in path.parts or ".venv" in path.parts or "site-packages" in path.parts:
            continue
        if path.name == "test_repo_hygiene.py":
            continue
        try:
            text = path.read_text(encoding="utf-8", errors="ignore").lower()
        except (PermissionError, OSError):
            continue
        if phrase in text:
            hits.append(str(path.relative_to(REPO_ROOT)))
    assert not hits, f"Killed phrase '{phrase}' reintroduced in: {hits}"
```
